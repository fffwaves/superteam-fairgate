◈ FAIRGATE
The Problem

Reputation is
invisible on-chain.

Web3 communities can't tell a whale from a bot, a long-term holder from a quick flipper, a DeFi native from a tourist. Trust is assumed — not earned.

Pain 01

No signal

Anyone can join any community. Wallet age, activity, and commitment are invisible to gating systems.

Pain 02

Easy to fake

Social metrics (followers, likes) are gameable. On-chain history is tamper-proof — but untapped.

Pain 03

No standard

Every project builds their own allowlist logic from scratch. There's no shared reputation layer.

Under the Hood

FairScale API —
six signals, one score.

FairGate calls the FairScale API server-side (key protected) and surfaces all six data points in the dashboard UI.

fairscore_base — on-chain activity score social_score — off-chain reputation tx_count — transaction volume active_days — consistency wallet_age_days — longevity lst_percentile — staking rank
